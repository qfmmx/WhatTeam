# NEPM-Admin 接口文档

> 东软环保公众监督平台 —— 管理端（nepm-admin）后端接口规范
>
> 本文档依据 `src/api/` 与 `src/mock/` 整理，描述前端实际调用的全部接口。
> 接通真实后端时，将 `src/config/index.js` 中的 `USE_MOCK` 置为 `false` 即可。

---

## 1. 基本约定

| 项目 | 说明 |
|---|---|
| BaseURL | `/api`（开发环境经 Vite 代理转发到 `http://localhost:8080`，见 `vite.config.js`） |
| 数据格式 | `Content-Type: application/json` |
| 超时 | 10000 ms |
| 字符编码 | UTF-8 |

### 1.1 统一响应结构

所有接口返回如下包装结构（见 `src/utils/request.js` 拦截器）：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": { }
}
```

| 字段 | 类型 | 说明 |
|---|---|---|
| code | number | 业务状态码，`200` 为成功；非 `200` 前端会弹出 `message` 并 reject |
| message | string | 提示信息 |
| data | any | 业务数据，结构因接口而异 |

> 响应拦截器在 `code === 200` 时返回整个 `{code,message,data}` 对象；调用方通过 `res.data` 取业务数据。

### 1.2 分页结构

凡分页接口，`data` 为如下结构：

```json
{
  "total": 9,
  "pages": 1,
  "current": 1,
  "size": 10,
  "records": []
}
```

| 字段 | 类型 | 说明 |
|---|---|---|
| total | number | 总记录数 |
| pages | number | 总页数 |
| current | number | 当前页码 |
| size | number | 每页条数 |
| records | array | 当前页数据列表 |

### 1.3 常用枚举

| 枚举 | 取值 | 含义 |
|---|---|---|
| 反馈状态 `state` | `0` | 待指派 |
| | `1` | 已指派（处理中） |
| | `2` | 已完成 |
| 是否超标 `isExceed` | `0` | 未超标 |
| | `1` | 超标 |
| 网格员状态 `status` | `1` | 启用 |
| | `2` | 禁用 |
| AQI 等级 `level` | `1`~`6` | 优 / 良 / 轻度污染 / 中度污染 / 重度污染 / 严重污染 |

## 2. 管理员模块

前端文件：`src/api/admins.js`

### 2.1 管理员登录

| 项 | 值 |
|---|---|
| 方法 | `POST` |
| 路径 | `/admins/login` |
| 前端方法 | `login(data)` |

**请求体**

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| adminCode | string | 是 | 管理员登录编码（mock 示例：`admin001`） |
| password | string | 是 | 登录密码（mock 示例：`123456`） |

**响应 data**

| 字段 | 类型 | 说明 |
|---|---|---|
| id | number | 管理员 ID |
| adminCode | string | 登录编码 |
| adminName | string | 管理员姓名 |
| role | string | 角色（如 `super`） |
| status | number | 状态 |

```json
{
  "code": 200,
  "message": "登录成功",
  "data": { "id": 1, "adminCode": "admin001", "adminName": "系统管理员", "role": "super", "status": 1 }
}
```

> 编码或密码错误时返回业务错误（前端弹出「编码或密码错误」）。

## 3. AQI 反馈模块

前端文件：`src/api/aqiFeedback.js`

### 3.1 分页查询反馈

| 项 | 值 |
|---|---|
| 方法 | `POST` |
| 路径 | `/aqiFeedback/page` |
| 前端方法 | `pageAqiFeedback(data)` |

**请求体**

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| pageNum | number | 是 | 页码，从 1 开始 |
| pageSize | number | 是 | 每页条数 |
| state | number | 否 | 反馈状态过滤（0/1/2） |
| cityCode | string | 否 | 城市编码，模糊匹配 |
| telId | string | 否 | 反馈人手机号，模糊匹配 |
| startTime | string | 否 | 反馈起始时间（`YYYY-MM-DD HH:mm:ss`） |
| endTime | string | 否 | 反馈结束时间 |

**响应 data**：分页结构，`records` 元素字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| id | number | 反馈编号 |
| telId | string | 反馈人手机号 |
| cityCode | string | 城市编码 |
| aqiValue | number | 观测 AQI 值 |
| feedbackTime | string | 反馈时间 |
| state | number | 状态（0/1/2） |
| gmId | number \| null | 指派网格员 ID |
| gmName | string | 网格员姓名（已指派时存在） |
| assignTime | string | 指派时间（已指派时存在） |
| remark | string | 反馈描述 |

### 3.2 根据 ID 查询反馈详情

| 项 | 值 |
|---|---|
| 方法 | `GET` |
| 路径 | `/aqiFeedback/{id}` |
| 前端方法 | `getAqiFeedbackById(id)` |

**路径参数**：`id` —— 反馈编号。

**响应 data**：单条反馈对象，字段同 3.1 的 `records` 元素。记录不存在时返回 `404`。

### 3.3 指派网格员

| 项 | 值 |
|---|---|
| 方法 | `PUT` |
| 路径 | `/aqiFeedback/assign` |
| 前端方法 | `assignAqiFeedback(data)` |

**请求体**

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | number | 是 | 反馈编号 |
| gmId | number | 是 | 被指派的网格员 ID |

**响应 data**：`number`（受影响行数，成功为 `1`，message 为「指派成功」）。
指派后该反馈 `state` 变为 `1`，并写入 `gmName`、`assignTime`。

### 3.4 更新反馈状态

| 项 | 值 |
|---|---|
| 方法 | `PUT` |
| 路径 | `/aqiFeedback/state` |
| 前端方法 | `updateAqiFeedbackState(data)` |

**请求体**

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | number | 是 | 反馈编号 |
| state | number | 是 | 目标状态（0/1/2） |

**响应 data**：`number`（受影响行数，成功为 `1`）。

## 4. 统计模块

前端文件：`src/api/statistics.js`

### 4.1 分页查询统计记录

| 项 | 值 |
|---|---|
| 方法 | `POST` |
| 路径 | `/statistics/page` |
| 前端方法 | `pageStatistics(data)` |

**请求体**

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| pageNum | number | 是 | 页码 |
| pageSize | number | 是 | 每页条数 |
| cityName | string | 否 | 城市名称，模糊匹配 |
| isExceed | number | 否 | 是否超标（0/1） |
| startTime | string | 否 | 确认起始时间 |
| endTime | string | 否 | 确认结束时间 |

**响应 data**：分页结构，`records` 元素字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| id | number | 记录 ID |
| cityCode | string | 城市编码 |
| aqiValue | number | AQI 值 |
| confirmTime | string | 确认时间 |
| gmId | number \| null | 确认网格员 ID |
| isExceed | number | 是否超标（0/1） |

### 4.2 根据 ID 查询统计详情

| 项 | 值 |
|---|---|
| 方法 | `GET` |
| 路径 | `/statistics/{id}` |
| 前端方法 | `getStatisticsById(id)` |

**响应 data**：单条统计对象，字段同 4.1。记录不存在返回 `404`。

### 4.3 各省超标统计

| 项 | 值 |
|---|---|
| 方法 | `GET` |
| 路径 | `/statistics/provinceTotal` |
| 前端方法 | `getProvinceTotal()` |

**响应 data**：数组，元素字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| provinceName | string | 省份名称 |
| exceedCount | number | 超标记录数 |
| totalCount | number | 总记录数 |
| exceedRate | string | 超标率（如 `66.7%`） |

### 4.4 AQI 等级分布

| 项 | 值 |
|---|---|
| 方法 | `GET` |
| 路径 | `/statistics/aqiDistribute` |
| 前端方法 | `getAqiDistribute()` |

**响应 data**：数组（6 个等级），元素字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| level | number | 等级（1~6） |
| levelName | string | 等级名称（优/良/轻度污染/中度污染/重度污染/严重污染） |
| count | number | 该等级记录数 |
| percentage | string | 占比（如 `13.2%`） |

### 4.5 超标趋势（近 12 个月）

| 项 | 值 |
|---|---|
| 方法 | `GET` |
| 路径 | `/statistics/trend` |
| 前端方法 | `getAqiTrend()` |

**响应 data**：数组（按月），元素字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| month | string | 月份（`YYYY-MM`） |
| exceedCount | number | 当月超标数 |

### 4.6 概览指标接口

以下接口均为 `GET`，无参数，`data` 为标量值，用于首页/概览卡片：

| 前端方法 | 路径 | 返回 data | 说明 |
|---|---|---|---|
| `getAqiTotalCount()` | `/statistics/totalCount` | number | 统计记录总数 |
| `getAqiGoodCount()` | `/statistics/goodCount` | number | 优良（AQI ≤ 100）记录数 |
| `getProvinceCoverage()` | `/statistics/provinceCoverage` | string | 省份覆盖率（如 `20.6%`） |
| `getCityCoverage()` | `/statistics/cityCoverage` | string | 城市覆盖率（如 `15.1%`） |
